# Архитектура

## Сервер — источник истины

Клиент не начисляет награды и не изменяет баланс самостоятельно. Сервер хранит игроков, завершённые события, общие разломы, инвентарь, артефакты, лоты, сделки, Круги и экономические фонды.

## Единый мир

Мир делится на координатные ячейки. Идентификатор объекта определяется секретным `WORLD_SEED`, временной эпохой, координатами ячейки и номером слота. Поэтому все игроки получают одинаковые объекты, не сохраняя миллионы неактивных точек в базе.

Обычный объект имеет общий идентификатор, но персональное завершение. Разлом использует `worldProgress`: общий вклад и список участников.

## API

- `GET /api/bootstrap` — игрок, каталоги, рынок и Круг;
- `GET /api/world` — события рядом;
- `POST /api/heartbeat` — присутствие и проверка движения;
- `POST /api/encounters/start|resolve` — серверное испытание и награды;
- `POST /api/craft`, `/api/equip`, `/api/shelter/upgrade`;
- `GET/POST /api/market/*`;
- `GET/POST /api/circle/*`;
- `GET /api/stream` — события реального времени;
- `GET /api/crypto/status` — состояние будущего адаптера.

## Переход к production

1. PostgreSQL + PostGIS: игроки, предметы, лоты, артефакты, аудит.
2. Redis: presence, rate limits, временные испытания, Pub/Sub.
3. WebSocket gateway вместо одного SSE-процесса.
4. Фоновый сервис LiveOps для сезонов и мировых событий.
5. Очереди для транзакций рынка и блокчейн-моста.
6. Объектное хранилище для пользовательской графики и журналов.
7. OpenTelemetry, централизованные логи и алерты.
